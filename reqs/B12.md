# B12 — Hardcoded "Rack" node name regardless of variant

## Summary
A `mini` build still announces "Rack" on the network and in the TCP banner.
Cosmetic. Relates to **R20** (rack/mini variants). Documented as a known wart in
[info.md](../info.md).

## Where
- TCP banner ([src/main.c:546](src/main.c#L546)); ArtPollReply short/long name ([src/main.c:738-739](src/main.c#L738-L739)).

## Actual
The node name is hardcoded `"… Rack"` carried over from the original, so every
build reports "Rack".

## Expected
The name reflects the build variant.

## Severity
Low (cosmetic).

## Fix
- [ ] Derive the name from `VARIANT_NAME` (banner + ArtPollReply).

Tracked as T4. Note: R22 (native Art-Net config) also touches the node name.
