# R30 — Web UI node-name editing (suffix model)

**Requirement.** The web UI shall let the user edit a single node-name *suffix*,
from which the short name, long name, and hostname are derived by fixed prefixes.

## Rationale
The node carries three identities — Art-Net ShortName, Art-Net LongName, and (R34)
the network hostname. Editing them independently was redundant and error-prone: in
practice they only ever differ by one trailing token (the node's location, e.g.
`Rack`). Consolidating to one editable suffix keeps all three consistent by
construction and gives the user a single field to fill in. Traces to **G6**
(field-serviceable over the network).

## Behavior (as-built)
- The user edits only the **suffix** (e.g. `Rack`). The node derives:
  - short name = `"LF " + suffix`        → Art-Net ShortName (field cap 17 + null)
  - long name  = `"LICHTFETISCH ArtNet Node " + suffix` → Art-Net LongName (cap 63 + null)
  - hostname   = `"LF-ArtNetNode-" + suffix` (sanitized to a DNS label — **R34**)
- The suffix is capped at **14** chars (so `"LF "` + 14 = 17, the ShortName budget).
  Long name and hostname have ample room.
- The browser **tab title** shows the derived short name; the page **header** shows
  the derived long name. Both refresh on every `/api/state` poll. A pencil icon
  opens a small dialog with one **Name suffix** field (maxlength 14) and a live
  read-only preview of the resulting short/long/host names. Save commits; Cancel
  closes.
- `POST /api/config` accepts a single `suffix` query param. httpd does not URL-decode
  query values, so it is `url_decode()`d in place (`%XX` / `+`), `strncpy`-capped to
  14, and persisted via `save_node_name()` (shared NVS path, key `SUFFIX`). On save
  the node re-derives all three names and re-applies the hostname.
- `/api/state` reports `suffix`, the derived `name` (short), `lname` (long), and
  `host` (hostname).
- Default suffix is the build `VARIANT_NAME` (`rack`/`mini`), so a fresh node still
  announces its variant (B12).

## Notes
- **Art-Net name writes removed.** ArtAddress (R22) no longer sets the node name — a
  controller cannot override the derived identity; the suffix is web-UI-only. The
  ArtPollReply still *reports* the derived short/long names.
- **NVS migration.** Old nodes stored `SHORTNAME`/`LONGNAME`; these keys are no
  longer read. An upgraded node reverts to the default (`VARIANT_NAME`) suffix until
  the user sets a new one. The dead keys are harmless.

## Implementation
- Firmware: `apply_node_names()`, `load_node_name`/`save_node_name`,

- UI: name dialog + `openName()`/`saveName()`, title/header wiring in
