# R30 — Web UI node-name editing

**Requirement.** The web UI shall let the user view and edit the node short name
and long name.

## Rationale
The node name was only settable over Art-Net (ArtAddress, R22). A user configuring
the node from the web UI had no way to set its identity. Traces to **G6**
(field-serviceable over the network).

## Behavior (as-built)
- The browser **tab title** shows the short name; the page **header** shows the long
  name. Both refresh on every `/api/state` poll.
- A pencil icon next to the header opens a small dialog with Short name (maxlength
  17) and Long name (maxlength 63) fields, seeded from the last poll. Save commits;
  Cancel closes.
- `POST /api/config` accepts `sname` and `lname` query params (any subset). httpd
  does not URL-decode query values, so each is `url_decode()`d in place
  (`%XX` / `+`), then `strncpy`-capped to 17 / 63 — the same caps as the ArtAddress
  path — and persisted via `save_node_name()` (the shared NVS path, keys
  `SHORTNAME` / `LONGNAME`).
- `/api/state` reports both names: `name` (short) and `lname` (long).

## Implementation
- Firmware: `config_post_handler`, `url_decode`, `build_state_json` in
  [src/main.c](../src/main.c).
- UI: name dialog + `openName()`/`saveName()`, title/header wiring in
  [src/index.html](../src/index.html).
