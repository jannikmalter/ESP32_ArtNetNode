# R23 — Minimal web UI replacing the TCP CLI

**Requirement.** The node shall replace the TCP CLI with a minimal web UI exposing
the same settings and live state.

## Rationale
A web page is friendlier than a raw TCP line protocol for field configuration.
Traces to **G1**. Replaces R4/R17. Must use only ESP-IDF `esp_http_server` to keep
**R7** (no external libraries).

## Acceptance criteria
All must hold before R23 is marked Done in `reqs.md`.

- [ ] One config page exposes patch, sync, sync universe, and channels/output.
- [ ] Live state (refresh rate, current settings) is shown and refreshes.
- [ ] Settings changes reuse the `stopDMX()` → write NVS → `startDMX()` handshake.
- [ ] Implemented with `esp_http_server` only — no external libraries (R7).
- [ ] Decision recorded on whether the raw TCP CLI is removed or kept.

## Design notes
- Reuse the existing NVS handshake; never mutate shared DMX state while the generator is live.

## Open questions
- Remove the TCP CLI (R4/R17) once the web UI lands, or keep both?
