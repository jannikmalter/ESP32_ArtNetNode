# R23 — Interactive web UI replacing the TCP CLI

**Requirement.** The node shall replace the TCP CLI with an interactive web UI
exposing the same settings and live state. The UI shall be lightweight,
fast-loading, and visually polished (quality split out as **R27**).

## Rationale
A web page is friendlier than a raw TCP line protocol for field configuration.
Traces to **G6**. Replaces R4/R17. Must use only ESP-IDF `esp_http_server` to keep
**R7** (no external libraries).

## Acceptance criteria
All must hold before R23 is marked Done in `reqs.md`.

- [x] One config page exposes patch, sync, sync universe, and channels/output.
- [x] Live state (refresh rate, current settings) is shown and refreshes.
- [x] Settings changes reuse the `stopDMX()` → write NVS → `startDMX()` handshake.
- [x] Implemented with `esp_http_server` only — no external libraries (R7).
- [x] Decision recorded on whether the raw TCP CLI is removed or kept.

### R27 (quality) acceptance criteria
- [ ] All assets served from flash; no external CDNs, fonts, or JS/CSS libraries (R7).
- [ ] Page is small (single self-contained HTML; aim well under ~64 KB total) and
  loads fast on the device's `esp_http_server`.
- [ ] Layout is clean and responsive — usable from a phone in the field, not just desktop.

## Design notes
- Reuse the existing NVS handshake; never mutate shared DMX state while the generator is live.
- "Light, fast, beautiful": prefer a single hand-rolled HTML/CSS/JS page embedded in
  flash over any framework; inline assets to avoid extra round-trips. No build
  toolchain dependency creep (keep R7/R4 no-libs intact).
- The web UI is also the entry point for OTA triggered from the browser ([R26](R26.md)).

## Decisions
- **TCP CLI removed (2026-06-26).** The web UI covers all of R4/R17's settings, so
  the raw TCP CLI (`tcp_task`, port 1337) was deleted rather than kept alongside.
  R4/R17 retired (struck in [reqs.md](../reqs.md)).
