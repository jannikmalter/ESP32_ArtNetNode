# R24 — Rework DMX sync into a "new data" flag

**Requirement.** The node shall rework DMX sync into a "new data" flag with
clear-before-send semantics: set the flag when a sync-universe ArtDMX packet is
processed or an ArtSync packet (`0x5200`) arrives; `dmx_task` waits for the flag
and clears it *before* sending each frame, so a packet arriving mid-frame is not
lost.

## Rationale
Reworks **R13**. The current `trigger` mechanism stalls in BREAK
([src/main.c:244-246](src/main.c#L244-L246)) and clears the trigger only *after*
the frame ([src/main.c:299](src/main.c#L299)), so a packet that arrives late in a
frame is dropped. Traces to **G2** (deterministic, reliable output).

## Acceptance criteria
All must hold before R24 is marked Done in `reqs.md`.

- [x] A "new data" flag is set when a sync-universe ArtDMX packet is processed. (`trigger = 1`, [src/main.c](../src/main.c) OP_DMX branch)
- [x] The flag is also set when an ArtSync packet (OpSync `0x5200`) arrives, handled in `eth_task`. (OP_SYNC branch; gated on `synchronize`)
- [x] `dmx_task` waits for the flag and clears it at frame start (before send), not after. (frame-wrap check: `trigger=0; curbit=0;`)
- [x] A packet arriving mid-frame sets the flag and is reflected in the *next* frame (not lost).
- [x] The ~0.2 s failsafe timeout is preserved so the generator never hangs without sync. (`xthal_get_ccount() - last_frame < 50000000`)

## Status

**Done.** The flag was kept named `trigger` (not renamed) — by decision, the name
is fine. Clear-before-send behavior verified on hardware. ArtSync (`0x5200`) trigger
added alongside; it shares the same flag and is consulted only in sync mode.

## Design notes
- Adds ArtSync (OpSync `0x5200`) handling in `eth_task`.
- Replaces the `trigger`/stall-in-BREAK pattern; keep the failsafe timeout.
