# R22 — Native Art-Net output configuration and query

**Requirement.** The node shall support native Art-Net output configuration and
query: set the per-output universe patch and node name from a console via
ArtAddress (`0x6000`), and report current config in ArtPollReply.

## Rationale
Replaces the current `0x6000`/`0x7000` ArtPoll-trigger quirk
([src/main.c:724](src/main.c#L724)), so standard Art-Net controllers can configure
the node without the raw TCP CLI. Traces to **G1**. Touches B12 (node name).

## Acceptance criteria
All must hold before R22 is marked Done in `reqs.md`.

- [ ] ArtAddress (`0x6000`) sets the per-output universe patch and persists it (NVS).
- [ ] ArtAddress can set the node name and persists it.
- [ ] ArtPollReply reports current per-port universes, name, and status.
- [ ] Optional: ArtInput (`0x7000`) handled.
- [ ] The old `0x6000`/`0x7000` ArtPoll-trigger quirk is removed.

## Design notes
- Field layouts in [docs/art-net.pdf](../docs/art-net.pdf).
- Reuse the existing `stopDMX()`/`startDMX()` + NVS handshake for any persisted change.
