# R34 — Network hostname derived from the name suffix

**Requirement.** The node shall set its network hostname, derived from the name
suffix (R30) by a fixed prefix and sanitized to a valid DNS label.

## Rationale
The firmware previously never set a hostname, so the node appeared on the LAN/DHCP
as the lwIP default `espressif` — useless for finding a specific node among many.
A stable, location-named hostname (`LF-ArtNetNode-Rack`) lets the node be found by
name. Traces to **G6** (field-serviceable over the network).

## Behavior (as-built)
- Hostname = `"LF-ArtNetNode-" + sanitized(suffix)`.
- **Sanitization** (DNS labels allow only `[A-Za-z0-9-]`, must not end in `-`):
  every byte of the suffix that is not alphanumeric becomes `-`; any trailing `-`
  is trimmed. An empty suffix yields `LF-ArtNetNode` (the trailing `-` from the
  prefix is trimmed). Length is bounded by the buffer (64) and in practice by the
  14-char suffix cap, well under the 63-char DNS-label limit.
- Applied via `esp_netif_set_hostname(eth_netif, ...)` at boot (before the
  Ethernet/DHCP start) and re-applied whenever the suffix changes. A change takes
  effect on the next DHCP DISCOVER/renew or reboot — adequate for a between-shows
  config change.

## Acceptance criteria
- [x] Fresh node reports hostname `LF-ArtNetNode-<variant>` over DHCP, not `espressif`.
- [x] A suffix with spaces/punctuation still produces a valid DNS label.
- [x] Empty suffix produces `LF-ArtNetNode` (no trailing hyphen).

## Implementation
- `apply_node_names()` builds `node_hostname` and calls `esp_netif_set_hostname`;
  the netif handle is kept in a file-scope global. Boot apply is after
  `esp_netif_new`; the change path runs from `save_node_name()`. See
  [src/main.c](../src/main.c).
