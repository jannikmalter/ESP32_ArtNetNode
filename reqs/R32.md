# R32 — Frame-perfect sync output via double buffer

**Requirement.** In a sync mode (universe-sync or ArtSync) the node shall present
each DMX output frame as a single consistent snapshot, so an Art-Net update applied
mid-frame cannot tear the frame being generated.

## Rationale
Builds on **R24**. `dmx_task` reads `DMXbuf` one channel at a time across the whole
~22 ms frame ([src/main.c](../src/main.c) channel read). With a single buffer, an
ArtDMX `memcpy` from `eth_task` (core 0) lands mid-frame while `dmx_task` (core 1) is
reading — some channels old, some new = a torn frame. R24's flag fixes *which* frame
new data appears in; it does not stop the buffer being mutated under the generator.
Traces to **G2** (deterministic, reliable output).

## Design
- Add a second `NUM_OUT*512` back buffer `ArtSyncBuf` alongside `DMXbuf`.
- **Free-run** (`synchronize==0`): `eth_task` writes ArtDMX straight into `DMXbuf`
  (unchanged behavior — tearing is acceptable when free-running).
- **Sync** (`synchronize==1`): `eth_task` writes ArtDMX into `ArtSyncBuf` and sets
  `trigger` on the sync-universe ArtDMX or on ArtSync (`0x5200`).
- **The copy is done by `dmx_task`, not `eth_task`** — at the frame wrap, when it
  consumes `trigger`, before `curbit=0`: `memcpy(DMXbuf, ArtSyncBuf, NUM_OUT*512)`.
  Doing it in the generating thread, between frames (line idle), is the only place
  the copy provably cannot tear the frame. `Cold` is re-read after the copy so its
  duration does not shorten the next BREAK.

## Acceptance criteria
- [x] A second `ArtSyncBuf` back buffer is allocated (`NUM_OUT*512`), cleared on repatch.
- [x] In sync mode ArtDMX writes the back buffer; in free-run it writes `DMXbuf` directly.
- [x] On commit `dmx_task` copies the whole back buffer to `DMXbuf` at the frame boundary.
- [x] Free-run output path is byte-identical to before (no copy, no `Cold` reset).
- [x] Builds clean.

## Caveats (by design, acceptable)
- A torn *source* read (ArtDMX writing `ArtSyncBuf` during the copy) can land one
  channel a frame early — a valid byte either way, no bounds risk.
- A controller using ArtSync that also sends the `sync_addr` universe commits early
  on that universe; for clean ArtSync point `sync_addr` at an unused universe.

## Status
Built clean. Clear-before-send (R24) verified on hardware; the double-buffer snapshot
itself is not yet hardware-confirmed.
