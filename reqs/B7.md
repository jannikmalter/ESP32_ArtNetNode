# B7 — recvfrom and socket errors unhandled

## Summary
Socket and `recvfrom` error returns are not handled in `eth_task`. Violates **R21**
(bounds-checked, signed-safe ingest) and the robustness behind **R2**.

## Where
- `eth_task`. `recv_len` declared `uint32_t` ([src/main.c:667](src/main.c#L667)), socket create ([src/main.c:677](src/main.c#L677)), recvfrom and length check ([src/main.c:690-691](src/main.c#L690-L691)).

## Actual
`recv_len` is unsigned, so a `-1` from `recvfrom` becomes `0xFFFFFFFF`, passes the
`>= 10` check, and a stale `ArtNetBuf` is processed. The socket fd `s` is also an
unchecked `uint32_t`; if `socket()` fails the loop becomes a tight CPU spin.

## Expected
A failed `recvfrom` is detected and skipped; a failed `socket()` is detected
instead of spinning.

## Severity / masked
Medium. Masked: sockets normally succeed in deployment.

## Fix
- [ ] Use a signed length for `recvfrom`.
- [ ] Check `socket()` and `recvfrom()` returns; continue on error.

Tracked as T2.
