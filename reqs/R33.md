# R33 — ArtSync as a distinct third sync mode

**Requirement.** The node shall offer three selectable DMX sync modes — free-run,
universe-sync, and ArtSync — where each mode commits a frame on exactly one signal,
so the modes do not interfere.

## Rationale
R24 made an ArtSync (`0x5200`) packet *also* set the trigger whenever sync was on,
sharing the single `synchronize` flag with universe-sync. That coupling meant a
controller using ArtSync that also sent the `sync_addr` universe would commit early
on that universe. Making ArtSync its own mode removes the overlap and matches how
controllers actually pick one synchronisation method. Traces to **G2**.

## Mode model
`synchronize` ∈ `{SYNC_OFF=0, SYNC_UNI=1, SYNC_ART=2}` ([src/main.c](../src/main.c)):

| Mode | ArtDMX write dest | Commit signal | Back-buffer snapshot (R32) |
|------|-------------------|---------------|----------------------------|
| `SYNC_OFF` free-run | `DMXbuf` (live) | none — free-runs | no |
| `SYNC_UNI` universe-sync | `ArtSyncBuf` | ArtDMX for `sync_addr` | yes |
| `SYNC_ART` ArtSync | `ArtSyncBuf` | ArtSync `0x5200` | yes |

- Write dest is the back buffer for *both* sync modes (`synchronize != SYNC_OFF`).
- `dmx_task` waits for `trigger` and snapshots the back buffer for both sync modes.
- An ArtSync packet sets `trigger` only in `SYNC_ART`; a `sync_addr` ArtDMX sets it
  only in `SYNC_UNI`. Free-run ignores both.

## Persistence & interfaces
- NVS `SYNC_STATE` (u8): values 0/1 keep their old meaning, so existing nodes are
  unaffected; 2 is the new ArtSync mode. Loaded value clamped to 0..2.
- Web `/api/config` `sync=0|1|2` (parsed + clamped); `/api/state` reports `sync`.
- Web UI Mode dropdown: Free-run / Uni-Sync / ArtSync ([src/index.html](../src/index.html)).
- Art-Net patch via ArtAddress still resets to `SYNC_OFF` (mode can't be set over Art-Net).

## Acceptance criteria
- [x] `synchronize` is a 3-value mode with named constants.
- [x] Both sync modes write the back buffer and snapshot at the frame edge.
- [x] ArtSync commits only in `SYNC_ART`; `sync_addr` ArtDMX commits only in `SYNC_UNI`.
- [x] Config/NVS parse and clamp the mode; web UI exposes all three.
- [x] Builds clean.
